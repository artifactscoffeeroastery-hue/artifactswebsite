Critical

Discount code silently disappears on delivery change

When the user switches from PUDO Locker to Local Overnight, the applied FOUNDER20 code is cleared with no warning. The total jumps from R790 to R872 — a R82 increase. Users who don't notice are overcharged; users who do notice must re-enter the code, risking abandonment.

Cart → Delivery
High

Manual rate refresh required

"REFRESH DELIVERY RATES" is a button the user must tap. Delivery rates should recalculate automatically when address fields change. This adds cognitive overhead and suggests the system isn't responsive.

Cart
High

Ambiguous "delivery details required" warning

After switching to home delivery, the page shows "Delivery details required for courier booking." The form appears filled. The message doesn't tell the user what's missing or what action to take.

Cart
High

Promo banner text clipped on mobile

The FOUNDER20 banner is cut off — "se at checkout for R20 off your first order" is visible, but the beginning is hidden off-screen. New users can't read the full value proposition without scrolling sideways.

Cart
Medium

6-field delivery form with no autocomplete

Phone, province, street address, suburb, city, and postal code are all separate manual inputs on mobile. Google Places / address autocomplete would collapse this to 1–2 taps.

Cart
Medium

No sticky order total while scrolling

The cart page is long. As the user scrolls through delivery fields, the order total and CTA button scroll out of view. A sticky footer with total + "Pay" keeps the goal visible.

Cart
Medium

Collect vs. deliver not clearly separated

PUDO Locker (collect) and courier options (deliver to address) are shown in the same flat list. Different flows — one needs no address, one needs 6 fields — should be separated visually (e.g., tabs or segmented control).

Delivery Picker
Low

Product page defaults to smallest size

80g (R75) is pre-selected, but specialty coffee buyers often want 200g or 1kg. Defaulting to the most popular size reduces add-to-cart friction.

Product Page
Low

Scarcity indicator lacks context

"17 spots left" implies limited stock but it's unclear if this is per bag, per batch, or per week. Specificity ("17 bags of this roast left") increases trust and urgency.

03 — Recommendations

6 Changes, Ranked by Impact

Fix 01 · Critical

Persist discount codes across delivery changes

Store the applied promo code in cart state. When delivery method changes, re-validate and re-apply the code automatically. If the code becomes invalid for the new method, show an inline message — never silently clear it.

Revenue protection Trust
Fix 02 · High

Auto-refresh delivery rates on field blur

Trigger a rate recalculation whenever the province or postal code field loses focus. Eliminate the manual refresh button. Show a spinner inline if rates are loading.

Reduced friction Fewer steps
Fix 03 · High

Split collect / deliver into tabs

Use a two-tab layout: "Collect (PUDO)" and "Deliver to me". Selecting Collect hides address fields entirely. Selecting Deliver shows the form. This cuts visual complexity and sets clear expectations.

Clarity Form reduction
Fix 04 · High

Fix promo banner overflow on mobile

Constrain the banner to viewport width. Use text wrapping or a marquee only if the text is intentionally animated. The full offer must be readable without horizontal scrolling.

Conversion Brand
Fix 05 · Medium

Add address autocomplete

Integrate Google Places API or a South African address lookup (e.g. AfriGIS) on the street address field. Auto-populate suburb, city, and postal code. Reduce 6 manual fields to 2–3 taps.

Mobile UX Speed Accuracy
Fix 06 · Medium

Add a sticky checkout footer

Pin a footer to the bottom of the cart page showing: item count, current total, and a "Pay Securely" button. As the user scrolls through delivery options, the goal remains visible and one tap away.
